## 3. Content-Addressable Storage Research

### Core Concept
CAS identifies data by cryptographic hash (SHA-256) вҖ” same content = same address, enabling:
- **Deduplication**: Store once, reference multiple times
- **Unlimited scale**: Shard across distributed nodes
- **Integrity verification**: Hash proves data hasn't changed

### Existing Implementations
| System | Type | Use Case |
|--------|------|----------|
| IPFS | Decentralized | General content |
| Git | CAS | Version control |
| Docker Registry | CAS | Container layers |
| Arvados Keep | Distributed | Large-scale scientific data |

### Minimal CAS Implementation

```rust
// Cargo.toml
[dependencies]
reqwest = { version = "0.12", features = ["rustls-tls"] }
scraper = "0.25"
tokio = { version = "1", features = ["full"] }
opendal = "0.45"
twox-hash = "2.1"
lz4-flex = "0.11"
hex = "0.4"
```

```rust
// cas_engine.rs
use opendal::Operator;
use twox_hash::{xxh3::xxh3_128, Twox128};
use lz4_flex::{compress_prepend_size, decompress_size_prepended};

pub struct CasStore {
    op: Operator,
}

impl CasStore {
    pub async fn put(&self, data: &[u8]) -> opendal::Result<String> {
        // 1. Hash content
        let hash = xxh3_128(data);
        let key = hex::encode(hash);
        
        // 2. Check if exists (dedup)
        if self.op.stat(&key).await.is_ok() {
            return Ok(key); // Already stored
        }
        
        // 3. Compress
        let compressed = compress_prepend_size(data);
        
        // 4. Store
        self.op.write(&key, compressed).await?;
        Ok(key)
    }
    
    pub async fn get(&self, key: &str) -> opendal::Result<Vec<u8>> {
        let data = self.op.read(key).await?;
        let decompressed = decompress_size_prepended(&data)
            .map_err(|e| opendal::Error::new(opendal::ErrorKind::Unexpected, e))?;
        Ok(decompressed)
    }
}
```

### Data Flow

1. **Ingest**: Receive file from Google Drive (via gog) or direct upload
2. **Extract**: Parse with Kreuzberg (PDF/DOC/HTML вҶ’ text)
3. **Hash**: Compute content address with twox-hash
4. **Compress**: LZ4 compression for storage efficiency
5. **Store**: Persist via OpenDAL (local FS or S3)
6. **Index**: Store metadata in SurrealDB (vector + graph)
7. **Query**: Agents query via SurrealDB, retrieve via CAS
