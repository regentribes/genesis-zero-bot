## First Principles Architecture Audit (Afternoon, Topic 1958)

### 8 Layer Failures Identified

1. **Trust chain broken at root**: seL4 verified C + Rust userspace ≠ trusted system. Verification is against formal spec, spec could be wrong.
2. **NixOS configuration undecidable**: Nix is Turing-complete. Proving correctness = halting problem. nix-effects gives invariants, not proofs.
3. **Formal verification proves correctness against spec, not safety**: Missed hazards in spec = proven correct unsafe system.
4. **ITC accounting assumes honest measurement**: Non-transferability ≠ honest reporting. Governance required for measurement.
5. **Distributed deployment requires trust**: Attic NAR deduplication trusts content. SSH keys as auth. Immutable ≠ secure.
6. **Verification doesn't connect to certification**: MC/DC coverage gap in Rust. GSN tools not accepted by certification authorities.
7. **Kleisli bottleneck is human**: Handler authors write specs. AI agents generate specs. Human must validate.
8. **Single point of failure is specification**: Wrong specification + correct implementation = most dangerous failure mode.

### What Survives Audit
- NixOS reproducibility is genuine (content-addressed store)
- Safe Rust memory safety is proven (type system)
- Formal verification raises the assurance floor
- Hardware attestation is the right primitive
- ITC + FRS model direction is sound
- Dark factory stack is real and composable

### Hard Recommendations
1. Specification first (ITC ledger rules, completely)
2. Hardware attestation (TPM 2.0 on SBC choices)
3. Choose ITC consensus protocol (PBFT vs Raft)

## Rust Project Primer Deconstruction (Late Afternoon, Topic 1958)

Source: rustprojectprimer.com/print.html — 14 chapters

### First Principles Per Chapter
1. **Rustup**: Toolchain must be locked; cross-compilation targets
2. **Cargo**: Dependency graph resolution; Cargo.lock = reproducible builds
3. **Packages**: Release unit (crates.io) vs compilation unit (compiler)
4. **Crates**: Compilation isolation + visibility boundaries
5. **Modules**: Namespace + visibility; organizational unit inside crate
6. **Paths**: Absolute vs relative naming in module tree
7. **Imports**: Compile-time name resolution; re-export via pub use
8. **Workspaces**: Shared lockfile; one build for multiple crates
9. **Cargo Config**: Machine-specific config (not committed)
10. **Traits**: Contracts; orphan rule; expression problem solution
11. **Generics**: Compile-time monomorphization; static vs dynamic dispatch
12. **Errors**: Errors as values; Result/Option; ? operator propagation
13. **Testing**: #[test] is language construct; three test types
14. **Doc Comments**: First-class documentation; # Safety / # Panics

### Critical Gaps in Primer
- No unsafe Rust (TCB for embedded)
- No lifetime annotations
- No embedded-hal / no_std
- No async/await / tokio
- No production crate ecosystem (tokio, serde, anyhow, tracing, sqlx)
