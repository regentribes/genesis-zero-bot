## Safety-Critical Software Tools (2026-04-13 11:50 UTC)

### Three Tools Analyzed

**CBMC** (bounded model checker, C/C++):
- Single binary, zero external dependencies
- Bounded exhaustive state exploration via SAT/SMT solving
- Write assertions as specification; CBMC proves for all inputs
- AWS used it for critical C infrastructure
- Use for: C sensor code, embedded firmware, anything where you want exhaustive verification without a build system

**Kani** (Rust model checker, Amazon/Rust Foundation):
- `cargo install --locked kani-verifier && cargo kani setup`
- Rust plugin that bundles CBMC internally (no manual CBMC install)
- Checks all possible states via symbolic execution
- Excellent for: unsafe Rust blocks, proving no panics, arithmetic properties
- Not yet ISO 26262 qualified but actively used in industry
- For digital twin: use in CI for critical Rust functions

**Verus** (theorem proving for Rust):
- Extends Rust with specification language (requires/ensures clauses)
- Uses SMT solvers, can prove unbounded properties
- Requires modified Rust compiler (heavier install than Kani)
- Microsoft research: AutoVerus + VeruSAGE (LLM-assisted proof synthesis)
  - AutoVerus: 150 algorithm tasks, ~60% solve rate
  - VeruSAGE: 849 real-world systems tasks (OS kernels, KV stores), ~40% solve rate
- Source: `git clone https://github.com/verus-lang/verus.git`
- For digital twin: when inference core is in Rust and needs complex state verification

### MORK Dependency Assessment
- MORK is pure Rust at runtime (all crates are Rust)
- Optional eval crate uses dynamic C library loading for benchmarks only (feature-gated)
- Can disable C FFI entirely for pure Rust deployment
- pathmap is also pure Rust (Adam Vandervorst's project)

### Practical Architecture for Regen Digital Twin

- Kani for Rust verification at CI time (Cargo plugin, no runtime deps)

```python
# Water loop generative model (simplified)
# P(true_level | sensor_reading) proportional to P(sensor_reading | true_level) * P(true_level)
# Prior: Uniform from sensor min to max
# Likelihood: Gaussian(reading, sigma=2cm)
# Posterior: computed via Bayes theorem

class WaterGenerativeModel:
    def __init__(self, capacity, sigma_sensor=2.0):
        self.capacity = capacity  # cm
        self.sigma_sensor = sigma_sensor  # cm

    def prior(self, n_samples=1000):
        return np.random.uniform(0, self.capacity, n_samples)

    def likelihood(self, true_level, reading):
        return norm.pdf(reading, true_level, self.sigma_sensor)

    def posterior(self, prior_samples, reading):
        weights = self.likelihood(prior_samples, reading)
        weights /= weights.sum()
        return weighted_resample(prior_samples, weights)

    def predict_next(self, posterior, inflow_forecast, outflow_estimate):
        # Inflow/outflow in cm per hour
        predicted = posterior + inflow_forecast - outflow_estimate
        return np.clip(predicted, 0, self.capacity)
```
