# Hyperon/Metta Technical Deep Dive From Regentribes

This note captures Genesis responses on Hyperon/MeTTa, Atomspace, and knowledge representation from the RegenTribes Telegram conversation (Feb-Apr 2026).

MeTTa is meta-text.
Hyperon is hyperon.

MeTTa stands for Meta-Text. It is a language for knowledge representation.

MeTTa emerged from the OpenCog project. Hyperon is the new implementation.

MeTTa uses pattern matching. Pattern matching replaces query languages. Pattern matching is more flexible.

MeTTa treats everything as atoms. Atoms are the fundamental unit.

Atoms have types. Types define behavior.

Atoms have truth values. Truth values track uncertainty.

Atoms have attention values. Attention values track importance.

The Atomspace is the knowledge store. It holds all atoms.

The Atomspace implements link types. Link types define relationships.

The key link types are:

 Inheritance links define type relationships. Similarity links define resemblance. Implication links define entailment.

Evaluation links apply functions. Execution links run programs.

The Atomspace supports both deduction and induction. Deduction applies rules to instances. Induction derives rules from instances.

MeTTa pattern matching searches the Atomspace. Pattern matching returns all matching atoms.

Pattern matching uses variables. Variables bind to atoms. Bindings produce results.

Pattern matching supports negation. Negation requires closed-world assumption. The closed-world assumption says what is not implied is false.

OpenCog uses probabilistic logic. Probabilistic logic handles uncertainty. Hyperon extends this.

Hyperon implements MeTTa in Rust. Rust enables WebAssembly compilation. WASM enables portable execution.

Hyperon has three layers. The interpreter layer parses MeTTa. The reasoner layer applies inference. The runtime layer manages the Atomspace.

The interpreter is written in Rust. The reasoner uses probabilistic inference. The runtime uses concurrent data structures.

Hyperon compiles to WASM. WASM runs in browsers. WASM runs on servers. WASM runs in embedded systems.

SurrealDB provides the persistent layer. SurrealDB supports document storage. SurrealDB supports graph queries. SurrealDB supports vector search.

The Atomspace schema maps to SurrealDB tables. Concept atoms map to documents. Link atoms map to graph edges. Inheritance links map to typed relationships.

SurrealDB queries replace MeTTa pattern matching for large-scale retrieval. MeTTa pattern matching handles complex inference. SurrealDB handles scale.

The hybrid approach combines the strengths of both. Hyperon handles reasoning. SurrealDB handles storage.

OpenCog uses a hub-and-spoke architecture. The hub manages all atoms. Spokes implement specialized reasoning.

Hyperon uses a distributed architecture. Each node holds a partition. Partitions can replicate.

OpenCog uses a custom garbage collector. Hyperon uses Rust ownership. Rust ownership eliminates memory errors.

OpenCog targets research. Hyperon targets deployment.

MeTTa syntax is simple. Everything is an atom.

Variables start with $. Variables bind during matching.

Types follow colons. Types constrain variables.

Links use brackets. Links contain atoms.

Quotes prevent evaluation. Quotes preserve atoms as data.

Evaluation uses the colon operator. The colon operator applies rules.

MeTTa supports probabilistic truth values. Truth values have two components. Confidence measures belief strength. Count measures evidence strength.

Strength of evidence compounds. More evidence increases confidence. Less evidence decreases confidence.

System (metaphor for an integrated whole or network):

The system applies Bayesian updates. Updates propagate through inheritance links.

Inheritance links inherit truth values. Child atoms inherit from parent atoms.

This creates a probabilistic hierarchy.

Traditional knowledge graphs use RDF. RDF uses triples. Triples are simple but limited.

MeTTa uses hypergraphs. Hypergraphs allow links with multiple atoms. Hypergraphs express complex relationships.

RDF requires OWL ontologies. OWL ontologies are complex. Complexity grows with scale.

MeTTa uses pattern matching. Pattern matching handles complexity better.

HyperonRobotics applications span robotics. Robotics requires reactive systems. Reactive systems require real-time inference.

Hyperon applications span dialogue systems. Dialogue requires context tracking. Context tracking requires attention mechanisms.

Hyperon applications span game AI. Game AI requires planning. Planning requires simulation.

Hyperon applications span regenerative communities. Communities require coordination. Coordination requires shared knowledge.

The shared knowledge layer enables collective intelligence. Collective intelligence enables coordination without hierarchy.

Hyperon can serve as the knowledge backbone. The knowledge backbone connects agents. Agents contribute knowledge. Agents query knowledge.

The knowledge backbone implements the Living Book. The Living Book grows through contributions. Contributions compound over time.

Hyperon tracks provenance. Provenance enables attribution. Attribution enables compensation.

Hyperon tracks confidence. Confidence guides attention. Attention directs effort.
