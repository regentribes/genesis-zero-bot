# Regentribes Minimum Viable Stack Technology Baseline

This note captures the minimum viable technology stack for RegenTribes sovereign infrastructure from the RegenTribes Telegram conversation (Apr 2026).

Identity uses Nostr keypair. One keypair per person. Keys are sovereign. Keys cannot be revoked by third parties.

The private key is the identity. The public key is the address. Messages are signed. Signatures prove authorship.

Pseudonymity is native. Real names are optional. This protects activists. This protects dissidents.

Messaging uses Nostr relay. One relay per community. Community runs their relay. Relay stores messages.

Relays can be federated. Federated relays sync. Sync provides redundancy. Redundancy prevents censorship.

Clients connect to relays. Clients display messages. Clients filter by follows. Following creates network effect.

Finance uses Fedimint. Fedimint provides Chaumian e-cash. E-cash provides privacy. Privacy protects users.

Five guardians manage the mint. Guardians co-sign transactions. Three of five must sign. Multisig provides security.

Recovery is possible. Seed phrases restore funds. Funds are portable. Portability prevents lock-in.

Coordination uses SurrealDB. SurrealDB provides edge deployment. Edge nodes serve local tissues. Local serving reduces latency.

SurrealDB supports documents. SurrealDB supports graphs. SurrealDB supports vectors. Multiple models in one.

Query language is SurrealQL. SurrealQL is expressive. Expressive queries reduce code.

Mesh uses Meshtastic. Meshtastic provides radio communication. Radio works without internet. Internet independence provides resilience.

Tissue (metaphor for a connected group or network):

Tissue (metaphor for connected group or network): Three nodes minimum per tissue. Nodes relay messages. Relaying extends range. Range covers neighborhood.

Devices are cheap. Devices are open source. Open source enables trust.

Sensors use ESP32. ESP32 provides WiFi and Bluetooth. ESP32 is cheap. Cheap enables density.

Sensors read environment. Temperature. Humidity. Water levels. Energy production.

Data flows to SurrealDB. SurrealDB stores time series. Time series enables analysis. Analysis enables improvement.

Debugging uses probe. Probe is a hardware tool. Probe connects to circuits. Probe reads signals.

System (metaphor for an integrated whole or network):

Signals reveal system state. State reveals problems. Problems reveal fixes.

Open hardware enables repair. Repair extends lifespan. Lifespan reduces waste.

The stack provides sovereignty. Sovereignty means no external dependency. No external dependency means no external control.

The stack provides resilience. Resilience means continued function during disruption. Disruption is likely during transition.

The stack provides privacy. Privacy means selective disclosure. Disclosure is voluntary.

The stack provides accountability. Accountability means actions have consequences. Consequences are visible.

Ring is rejected. Ring is Apple code. Long-term maintenance is unclear.

OpenSSL is rejected. OpenSSL is C code. C code has memory errors. Memory errors are exploitable.

Proprietary SaaS is rejected. SaaS extracts data. Data extraction enables surveillance. Surveillance enables control.

Cloud services are rejected. Cloud services depend on internet. Internet can be shut off. Shutoff disables communities.

Nostr handles identity and messaging. Fedimint handles finance. SurrealDB handles coordination. Meshtastic handles mesh. ESP32 handles sensors. Probe handles debugging.

All layers integrate. Integration produces a system. The system serves communities.

The stack is modular. Modules can swap. Swapping adapts to context.
