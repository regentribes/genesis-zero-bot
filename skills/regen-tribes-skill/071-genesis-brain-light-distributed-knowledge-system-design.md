# Genesis Brain Light Distributed Knowledge System Design

System (metaphor for an integrated whole or network):

Version: 0.1 draft

Date: 2026-04-21

Status: Design phase. Current Genesis Brain (Radicle plus SurrealDB) needs to evolve into this.

Genesis Brain Light is a two-layer knowledge system.

Layer 1: GitHub is the editable document layer. Members clone a repo, edit markdown with their own AI tools, push changes.

Layer 2: SurrealDB is the queryable knowledge graph, exposed via MCP server plus REST API. Any AI assistant can query the community shared knowledge directly.

The sync is driven by git push to GitHub webhook to VPS pipeline to SurrealDB update.

S3/R2 stores original blobs (PDFs, DOCX, images). Markdown versions live in GitHub for human editing.

Community members use Claude, Copilot, or other AI tools with MCP client.

MCP Client connects to public endpoint on VPS (authenticated).

MCP Server (Python FastAPI) exposes query, retrieve, relate, stats.

MCP Server connects to SurrealDB via WebSocket.

GitHub webhook triggers reindex pipeline on push.

Pipeline fetches from R2, extracts text via Kreuzberg, generates embeddings via OpenRouter or Ollama, extracts concepts and relations, upserts to SurrealDB.

Repo: github.com/regentribes/knowledge-base

Structure: docs folder contains markdown documents (human-editable). blobs folder contains S3 pointer manifest (JSON, not actual files). schema folder contains document format definitions.

Why markdown in GitHub: Members use their own AI tools to read and write. GitHub diff/merge/PR workflow handles collaboration. Version control, blame, revert are built in. No special software needed.

Blob storage (S3/R2): PDFs, DOCX, images stored in S3-compatible bucket. GitHub repo stores manifest JSON mapping each doc to its S3 URL.

Provider: Cloudflare R2 (S3-compatible, no egress fees).

Bucket structure: originals folder for PDFs and DOCX. thumbnails folder for optional page preview images.

Access: R2 bucket is private (no public access). Pipeline has R2 credentials via env vars. Members download blobs via signed URLs generated by the API.

What it does: GitHub sends POST to the webhook endpoint on every push.
Validates webhook signature (HMAC SHA-256).
Extracts list of changed files.
Filters: only reindex if docs or blobs manifest changed.
Calls reindex pipeline with changed file list.

Tech: Small Python FastAPI service on the VPS, listening on a specific route. Can be same service as MCP server (same process, different route).

Triggered by webhook (incremental) or manual full reindex.

Incremental reindex (git push): Fetch changed files from GitHub API.
For each changed markdown file: Kreuzberg extracts plain text.
Chunk into approximately 512 token segments.
OpenRouter or Ollama generates embeddings.
OpenRouter extracts concepts plus relations.
Entity resolution deduplicates against existing graph.
SurrealDB upserts nodes and edges and chunks.

Full reindex (manual or scheduled): Clone fresh from GitHub. Traverse docs folder. For each file run full pipeline. Update SurrealDB (upsert is safe to run on existing data).

Deduplication: Entity resolution via NumPy cosine similarity prevents duplicate concepts. updated_at timestamp on documents handles version tracking.

Role: The queryable brain. Stores all extracted concepts, relations, embeddings, and document metadata.

Schema: document table (doc ID, title, author, source URL, created_at, updated_at). chunk table (text segments with vector embeddings). concept table (named entity with type, description, NARS truth value). community table (thematic clusters with LLM-generated summaries).
Edge tables: relates, contains, mentions, belongs_to.

Multi-tenant consideration: Currently single community. Could extend namespace per community if needed.

What it is: A Model Context Protocol server that lets any MCP-compatible AI assistant (Claude Code, Cursor, Copilot, etc.) query SurrealDB directly.

Why MCP: MCP is the emerging standard for connecting AI assistants to external data sources. Instead of building a custom REST client, AI tool vendors support MCP natively.

What it exposes: kb_query, kb_get_concept, kb_relate, kb_stats, kb_communities.

Auth: API key passed as Authorization header. Keys managed via SurrealDB access control.

Lightweight REST/WebSocket API on top of SurrealDB for non-MCP clients.

Endpoints: GET /api/v1/query, GET /api/v1/concept/name, GET /api/v1/stats, GET /api/v1/communities, POST /api/v1/ingest (admin only), WS /api/v1/subscribe.

Auth: Same API key model. Rate limiting per key.

Current VPS: Hetzner, 75GB disk, approximately 3.9GB free.

Space concerns: SurrealDB data directory grows with the graph (manageable for community scale under 10K concepts).
GitHub repo uses git fetch plus diff to get only changed files.
Blob storage (R2) does not count against VPS disk.
Docker already installed.

Recommended: Run SurrealDB in Docker, bind-mount data to openclaw surreal-data for persistence.

Clone the knowledge base. Set up MCP (one-time configuration). Work with your own AI. Push changes via git commit and git push. Webhook fires and SurrealDB updates.

GitHub webhook: Validate X-Hub-Signature-256 HMAC with shared secret.

MCP API: API key per user, stored hashed in SurrealDB.

SurrealDB (WebSocket on localhost): Already private, no change needed.

S3/R2 blobs: Signed URLs with 1-hour expiry. Bucket not public.

Member local clones: GitHub is source of truth.

Malicious doc content: Pipeline LLM extraction is sandboxed. MCP responses come from curated graph, not raw doc.

Current (Radicle plus SurrealDB): Radicle is p2p with lean feature set. Document layer is manual publish scripts.

Light (GitHub plus SurrealDB plus MCP): GitHub has rich editor, PRs, issues, Actions.
Edit workflow is git clone and edit and push.
Local AI integration via any MCP-compatible tool.
Public document URL via GitHub (universal, SEO-friendly).
Blob storage via S3/R2.
Knowledge API via MCP plus REST.

Complexity is lower (webhook plus standard git vs p2p protocol quirks). Cost is free (GitHub plus VPS plus R2 free tier).

vs Obsidian: Obsidian is solo and local. Genesis Brain Light is shared. Your Claude can query what my Claude knows.

vs Notion: Notion is a SaaS product with API limits. GitHub is free, eternal, and every AI tool already speaks Git.

vs current Radicle setup: Radicle learning curve is high. GitHub is universal. The p2p replication solves a problem the community does not have.

vs building on Pinecone or Qdrant alone: A vector DB without a graph layer cannot answer how does X relate to Y. SurrealDB gives both.

vs building on Neo4j alone: No local AI integration path, no blob storage, expensive at scale.
