# The Body Information Domain

Body (metaphor for the complete collective or organism):

The body is the whole planetary network.
All bioregions. All organs. All tissues.

Information at this scale needs standards.
Without standards nothing connects.

A standard is an agreement.
It says this format means this thing.

The knowledge commons holds standards.
It holds best practices. It holds failure reports.

Organ (metaphor for a functional subsystem or team):

Every organ contributes what it learns.
Every organ takes what others have learned.

The knowledge graph grows.
Each node is a concept. Each edge is a relation.

A concept has a name. A concept has a definition.
A concept has evidence. A concept has a confidence level. A concept has a source.

An edge has a from concept. An edge has a to concept.
An edge has a type. An edge has a strength of evidence.

The graph is queried.
A question comes in. The graph returns relevant concepts.

A decision support query example.
System (metaphor for an integrated whole or network):

What is the best water system for clay soil in a dry climate.
The graph answers with matching concepts. With connecting relations. With confidence levels.

Standards at the body level: data formats. Measurement units. Safety thresholds.

Data format standard uses JSON for sensor readings. Parquet for large datasets.

Measurement unit standard uses SI units.
Liters. Kilograms. Meters. Seconds. Celsius.

Safety threshold standard uses maximum contaminant levels.
These do not vary by region.

The knowledge commons is open.
Read by anyone. Write by verified contributors.

Each contribution is reviewed. Accuracy matters.

False information is flagged.
Repeated false information gets a contributor banned.

Old information gets flagged for review.
Standards evolve. Old numbers get replaced.

The commons runs on open source.
Code is open. Data is open.

No corporation owns the commons.
No government owns the commons.
The users own it.

The commons is funded by contribution.
Each organ contributes time or goods annually.

Governance falls to a council of bioregion representatives.
They set standards.

Standards changes go through proposal. Comment. Revision. Final vote.

The body sees patterns across bioregions.
It sees which water systems work in dry climates.
It sees which fail. It sees which soil methods work in clay. It sees which fail.

A desert organ learns from a tropical organ.
A cold organ learns from a warm organ.
The graph connects distant experiments.

The body runs on distributed architecture.
Each server holds a graph partition.
They sync with each other.

The body has no central brain.
Many nodes cooperate.
Query routing finds relevant nodes.

The body has memory.
It remembers what worked. It remembers what failed.
Old failures stay visible. They warn new experiments.

A bioregion starting a water project queries the graph.
It finds similar projects. It finds what worked. It finds what to avoid.
It saves time and resources.

The graph supports decisions. It does not make decisions.
People make decisions.
