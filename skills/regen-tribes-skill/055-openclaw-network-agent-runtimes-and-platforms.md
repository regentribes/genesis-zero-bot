# Openclaw Network Agent Runtimes And Platforms

OpenClaw is an open-source AI agent framework.
It is built with TypeScript and Node.js.
It runs autonomous agents across messaging apps, browsers, files, and shell commands.
50-plus integrations. Self-hosted on VPS or local infrastructure.

The Ecosystem (metaphor for system or network): OpenClaw ecosystem has multiple variants.
Each variant serves a different context.
Variants vary in capability. Size. Security. Deployment target.

OpenClaw is the infrastructure standard.
It provides channels. Memory. Persistence.
It runs the agent loop.
The loop reads context. Calls tools. Updates memory.

OpenClaw serves as the baseline framework.
Maximum flexibility.
50-plus integrations.
Self-hosted on VPS or local infrastructure.

IronClaw provides WASM sandboxing.
Rust-based. 2.8K stars.
Less than 10ms startup.
7.8MB memory footprint.
Real isolation. Five-layer defense.
PostgreSQL and pgvector hybrid search.

Moltis provides voice.
Rust-based. 1.3K stars.
Real isolation via WASM sandbox or containers per session.

ZeptoClaw provides embedded deployment.
Rust-based. 305 stars.
Real isolation. Low-resource focus.
Small devices. Low power.

HiClaw provides zero-trust security.
Audits all access.
Every inference step verifiable.
Audit trail is the definition of trustworthy.

NullClaw provides ultra-lightweight deployment.
Zig-based. 1.6K stars.
678KB binary.
Runs on five-dollar ARM boards.
One megabyte RAM.
GPIO support. Regex-based security.

MicroClaw provides platform breadth.
Many channels. Many integrations.
Connects to everything.

NanoClaw provides security isolation.
700 lines of code.
OS-level isolation via Docker or Apple Container.
Designed for security teams.
