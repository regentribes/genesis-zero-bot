# Formation Coding Language

Formation Coding Language (FCL) sends ideas through space as shapes that change over time.

FCL does not send text. FCL does not send images.
FCL sends a shape that changes over time.
The shape is the message.

Normal communication depends on symbols.
Symbols must be agreed upon in advance.
Sender and receiver must share the symbol set.
If they do not, communication fails.

FCL does not use symbols.
FCL uses geometry.
Geometry is universal.
All intelligent systems understand geometry.
Two points make a line. Three points make a triangle.
A triangle means the same thing to any observer.

Sender and receiver share a knowledge graph.
The graph has nodes. The graph has edges.
Nodes are concepts. Edges are relations.
Both parties must have the same graph structure.
If they do not, the formation cannot decode.
This is the foundation.

A node becomes a point.
An edge becomes a vector.
The vector has direction. The vector has length.
Direction encodes the relation type.
Length encodes the relation strength.
The formation shows the graph structure in space.

The formation changes over time.
One shape becomes another shape.
The change encodes sequence.
The timing of pulses encodes priority.
Morphing between shapes encodes transition.
Time adds a second dimension of meaning.

A formation can describe other formations.
This enables recursion.
Recursion enables self-reference.
Self-reference enables reflection.
This layer makes FCL theoretically complete.

An autonomous aerial vehicle (AAV) carries LEDs.
It forms shapes in the sky.
Together they form a formation.
A formation sends a message. The message is the shape.

A bird flock is a natural substrate.
No permission needed. Lower precision. Lower bandwidth.
Starlings demonstrate this works.

A holographic projection is a volumetric display.
No physical vehicles required.
Limited by display technology.

A digital screen is 2D.
The formation loses a dimension. Some information is lost.

Start with a knowledge graph.
Choose a traversal path through the graph.
Map each node to a point.
Map each edge to a vector between points.
Add temporal sequence.
Encode as formation parameters.
Send on the chosen substrate.

Detect the formation. Reconstruct the points and vectors.
Map the geometry back to the knowledge graph.
Trace the traversal path.
Output the concept sequence.
Verify the graph structure matches.

FCL is not a display technology. The formation is not a picture.

FCL is not steganography. There is no hidden message. The shape is the message.

FCL is not morse code. No agreed symbol set. Geometry carries meaning.

FCL is not an image format. The meaning is in the graph traversal. Not in pixels.
