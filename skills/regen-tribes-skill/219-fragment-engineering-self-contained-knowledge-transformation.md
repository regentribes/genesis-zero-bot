# Fragment Engineering Self Contained Knowledge Transformation

Fragment engineering transforms input into self-contained knowledge fragments using four distinct processing modes.

Each fragment is independent. It contains full context and does not require external references to be understood.

The output follows ASD-STE100 plain language rules. Every sentence is short, active, and precise.

Every concept is described by its first principles. No proprietary labels.

System (metaphor for an integrated whole or network):

Mode 1 is spec. It transforms features into executable specifications using command form. It states what a system must do.

Mode 2 is shape. It transforms concepts into structured representations for semantic systems. It focuses on machine-readable format.

Mode 3 is interpret. It analyzes concepts through dynamic interpretive layers discovered within the input itself.

Mode 4 is extract. It extracts self-contained fragments covering all concepts and relationships when no other trigger is present.

Spec produces imperative requirements. Commands use SHALL for requirements, VERIFY for validation, ENSURE for constraints.

Procedural sentences must not exceed 20 words. Commands are direct and unambiguous.

Example: the system shall verify user credentials against the user database and shall issue a session token upon successful verification while rejecting invalid credentials without revealing username hints.

The specification focuses on observable behavior. Internal implementation is not specified unless it affects external behavior.

Shape produces structured data in machine-readable formats.

Supported formats include MeTTa and Toon-LD.

MeTTa is a pattern matching language for knowledge representation. It uses type declarations and function definitions.

Toon-LD is Token-Oriented Object Notation for Linked Data. It reduces token count by 40 to 60 percent compared to JSON-LD.

Shape output sends each structured format as a separate message. Each message contains only the code block. No explanatory prose surrounds the structured data.

Interpret discovers unique interpretive dimensions from within the input itself.

No fixed categories are used.
The interpreter analyzes domain, history, cultural references, implicit assumptions, contradictions, gaps, unstated implications, emotional charge, temporal markers, spatial references, relational patterns, power dynamics, language quirks, and metaphor anchors.

For each distinct aspect found, one interpretive layer is produced.

A minimum of two layers is always produced. Maximum is unlimited.

Example interpretive layers from an iMeta conference page: centralized gatekeeping through abstract submission identifiers enabling blind review, knowledge hierarchy with AI prominence at top
and
infrastructure at bottom, temporal positioning where intelligent metaverse names the gap it aims to close, absence of explicit registration fees showing academia-dependent funding model.

Extract produces a series of independent fragments covering every concept and relationship in the source.

Each fragment is self-contained. It includes full context needed to understand it without external references.

Collectively, fragments contain every concept, relationship, and detail from the source. No detail is omitted. No invented detail is added.

All terms are defined within the fragment or removed if undefined.

All fragment output follows ASD-STE100 rules universally.

Procedural sentences are maximum 20 words. Descriptive sentences are maximum 25 words.

Periods end sentences. Em dashes are not used.

One idea per sentence. Active voice is preferred.

Articles are used wherever grammatically required. Noun clusters are limited to three words.

Step 1: detect the trigger phrase to identify the mode.

If no trigger is detected, default to extract mode.

Step 2: process according to the identified mode.

For spec: decompose into requirements and express as commands.

For shape: generate MeTTa patterns and build Toon-LD structure.

For interpret: discover unique dimensions from input context and generate layers dynamically.

For extract: decompose into concepts and generate fragments with full context.

Step 3: verify ASD-STE100 compliance.

Step 4: send each fragment as a separate message.

All modes distinguish between two entity types.

Self-evolving entities are living entities that transition through different stages of perception. They use different lenses over time. Examples: persons, communities, projects.

Static objects are things that evolve independently or under human intent without agency. Examples: natural phenomena, technical systems, buildings.

This distinction affects how fragments interpret and structure information. Living entities require temporal tracking. Static objects require state tracking.

When both MeTTa and Toon-LD are produced, they must be consistent.

MeTTa type must match Toon-LD @type.

Toon-LD predicate relationships must mirror MeTTa function arity.

Every entity must have an @id.

Shape-based partitioning activates automatically when data sparsity exceeds 30 percent.

Fragment engineering provides the community knowledge system.

Community documentation enters as input. Fragments emerge as structured knowledge.

Interpret mode is used for understanding community narratives and cultural patterns.

Extract mode is used for capturing member ideas and project documentation.

Spec mode is used for creating community governance specifications.

Shape mode is used for building the community knowledge graph data structures.
