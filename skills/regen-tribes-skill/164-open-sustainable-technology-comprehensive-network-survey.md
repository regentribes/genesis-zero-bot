# Open Sustainable Technology Comprehensive Network Survey

System (metaphor for an integrated whole or network):

Ecosystem (metaphor for an interconnected network):

Open Sustainable Ecosystem (metaphor for system or network): Technology (OST) is a directory and analysis of the open-source ecosystem for climate change, sustainable energy, biodiversity, and natural resources.
The mission is to democratize technological development and knowledge creation for preserving planetary stability.
OST maintains a Grist spreadsheet tracking 900+ projects across 14 major categories.
All metadata is provided by ecosyste.ms.
The project is verified Digital Public Good since 2025.

The ecosystem includes over 900 projects documented.
Categories range from renewable energy to biodiversity, climate modeling, and sustainable development.
Each category contains subcategories addressing specific domains.

This category targets the entire system design of solar energy systems.
Tools cover data acquisition, site selection, material simulation, yield forecasting, and quality control.

Key projects:

Project 1 is pvlib-python.
Simulates performance of photovoltaic energy systems.
Provides documented functions for PV modeling.
Industry standard for solar modeling.

Project 2 is pvfactors.
Open source view-factor model for diffuse shading and bifacial PV modeling.
Developed by SunPower.

Project 3 is bifacial_radiance.
Toolkit for ray-trace modeling of Bifacial Photovoltaics.
Developed by NREL.

Project 4 is feedinlib.
Contains implementations of photovoltaic models for electricity generation calculation.
Part of the oemof ecosystem.

Project 5 is rdtools.
Open source library for reproducible technical analysis of time series data from PV systems.
Developed by NREL.

Project 6 is solax.
Reads energy usage data from real-time API on Solax solar inverters.
Enables monitoring of solar installations.

Project 7 is LibreSolar.
Firmware for BMS boards, MPPT, and PWM charge controllers.
Open source hardware and firmware for solar systems.

Project 8 is OpenDTU.
Alternative open source software for communicating with Hoymiles solar inverters.
Replaces proprietary cloud with local control.

Project 9 is SOLECTRUS.
Alternative photovoltaic dashboard visualizing yield and consumption.
Open source monitoring solution.

Project 10 is solXpect.
Android app to forecast output of photovoltaic systems.
Uses Open-Meteo data.

Project 11 is InSPIRE.
Scripts and studies for agrivoltaics research.
NREL project on dual-use land for agriculture and solar.

Project 12 is PV_ICE.
Open-source tool to quantify Solar Photovoltaics Energy and Mass Flows in the Circular Economy.
Developed by NREL.

Enables community customization and control.

Principle 2 is transparency.
Open code enables verification of sustainability claims.
Transparent data pipelines build trust.
Open methodologies allow independent validation.

Principle 3 is collaboration.
Shared development reduces redundant effort.
Best practices spread across communities.
Collective improvement accelerates progress.

Principle 4 is local adaptation.
Open tools enable customization for local conditions.
Communities can adapt without waiting for vendor updates.
Local knowledge integrates with global best practices.

Challenge 1 is fragmentation.
Many isolated projects with limited interoperability.
Standards development is ongoing.
Data formats vary across tools.

Challenge 2 is maintenance.
Volunteer-maintained projects face sustainability risks.
Funding models are often uncertain.
Long-term maintenance requires institutional support.

Challenge 3 is quality.
Code quality varies across projects.
Documentation is often incomplete.
Security audits are rare.

Challenge 4 is adoption.
Communities lack technical capacity to use tools.
Training and support resources are limited.
Onboarding complexity discourages new users.
