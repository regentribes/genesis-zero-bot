| COMPETENCY AREA - SYSTEMS ENGINEERING MANAGEMENT: DECISION MANAGEMENT |  | SELF-ASSESSMENT COMPETENCE LEVEL :  |  | UNASSESSED |
| Description:
 Decision management provides a structured, analytical framework for objectively identifying, characterizing and evaluating a set of alternatives for a decision at any 
point in the life cycle in order to select the most beneficial course of action. |  |  |  |  |
| Why it matters:
 System development entails an array of interrelated decisions that require the holistic perspective of the Systems Engineering discipline. Decisions include selection of 
preferred solution at every level of the system, including technology option selection, architecture selection, make-or-buy decisions, strategy selection for maintenance, 
disposal. While some low value decisions can be made and recorded simply, key decisions which might affect the long-term success and delivery of the project’s de
sired value need to be controlled using a formalized decision management process. |  |  |  |  |
